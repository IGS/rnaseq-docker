Introduction


Welcome to Grotto, the pipeline launcher tool for the IGS RNASeq pipeline. This tool lets the user tweak the pipeline as he or she sees fit, as well as provide premade sample and component configuration files, which can be useful for repeat analyses. When a pipeline is launched, the pipeline can be monitored, and results can be downloaded in the form of a BDBag object. The user can also view recent pipelines by project as well.

Sample Info File page


This is the first page encountered when creating a new pipeline. The user has the ability to either input samples directly into the page, adding rows as needed, or can upload a pre-made sample info file (ending in .info).

Clicking 'Next' will proceed to the Pipeline Options page.

Pipeline Options page


This page is where pipeline options are configured for the pipeline. There is a toggle that lets the user select between a Prokaryotic run or a Eukaryotic run. Note that selecting some checkboxes will cause secondary checkboxes to pop up, depending on the combination of selected boxes.

Clicking 'Next' will proceed to the Config File page.

Config File page


This page is where each component for the new pipeline can be configured. The user has the ability to either provide configuration parameters directly onto the page, or can upload a pre-made config file (ending in .config)

If filling in manually, parameters with red asterisks are required, and the page will register an error if the user attempts to proceed with one of these parameters blank.

Also if filling in manually, it is important to pay attention to the colored circle for each component. Each color implies a certain meaning:

  • Green - Component parameters are fine as they are.
  • Yellow - Component has some parameters where the default value will work for the majority of people, but it may be wise to look over them.
  • Red - Component has parameters that are required and will require the user's attention.

Clicking 'Next' will proceed to the Summary page.

Summary page


This page is where the selected files, options, and parameters can be reviewed before submitting the pipeline. If there is an error, the "Edit" button will take the user back to that page to edit fields. There is also a "Download" link to download the sample.info, pipeline_options.txt or pipeline.config file for use in future pipelines

Clicking "Submit" will submit the pipeline and proceed to the Pipeline Status page for this pipeline if the submission is successful

Pipeline Status page


This page is where a pipeline can be monitored. The user can view the status and runtime of the pipeline as well as the status and runtimes for each component.

Recent Pipelines page


This page is where the user can select recent pipelines to view. After selecting a project code, the 10 most recent pipelines belonging to that user are then shown to select.
